Description

On last stage we handled the login. Now, on this stage, we want that after a successful login users get redirected to a menu screen in which they reach the main features offered by our app

The navigation graph will help us to implement the navigation between the screen of our app.

Objectives

  1. Add a Fragment named UserMenuFragment to the navigation graph
  2. Add an TextView to UserMenuFragment with id userMenuWelcomeTextView, the text should be "Welcome $username" in which $username is the same username used to login
  3. Add a Button to UserMenuFragment with id userMenuViewBalanceButton and text view balance
  4. Add a Button to UserMenuFragment with id userMenuTransferFundsButton and text transfer funds
  5. Add a Button to UserMenuFragment with id userMenuExchangeCalculatorButton and text calculate exchange
  6. Add a Button to UserMenuFragment with id userMenuPayBillsButton and text pay bills
  7. On the design tab of nav_graph make a connection from LoginFragment to UserMenuFragment. The code generated after making the connection should contain an action with id action_loginFragment_to_userMenuFragment and destination userMenuFragment
  8. After a successful login users should be redirected to UserLoginFragment. To make the navigation you can use findNavController().navigate(R.id.action_loginFragment_to_userMenuFragment). It is also possible to pass a Bundle as second argument of navigate method to pass arguments to userMenuFragment, which you can later retrieve with arguments?.getString(...)
  9. All requirements from stage1 are still valid, including toast messages
