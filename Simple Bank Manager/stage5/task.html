Description

After successful login, user should be able to go to Transfer funds screen and transfer money to or from account, and check current balance at the View Balance screen.

The navigation graph will help us to implement the navigation between the screens of our app.

Objectives

  1. Add a Fragment named ViewBalanceFragment to the navigation graph
  2. Add an TextView to ViewBalanceFragment with id viewBalanceTextBalanceTextView, the text should be "Balance:"
  3. Add an TextView to ViewBalanceFragment with id viewBalanceShowBalanceTextView, with hint "0.0$". This TextView will be used to show account balance.
  4. On the design tab of nav_graph make a connection from UserMenuFragment to ViewBalanceFragment. The code generated after making the connection should contain an action with id action_userMenuFragment_to_viewBalanceFragment and destination viewBalanceFragment
  5. Add a Fragment named TransferFundsFragment to the navigation graph.
  6. On the design tab of nav_graph make a connection from UserMenuFragment to TransferFundsFragment. Code generated after making connection should have action with id action_userMenuFragment_to_transferFundsFragment and destination id transferFundsFragment
  7. Add to FragmentLayout TransferFundsFragment at the left upper side of the Fragment layout TextView with name transferFundsFromAccountTextView with text Transfer from.
  8. add to FragmentLayout TransferFundsFragment to the right upper side of FragmentLayout this component: FrameLayout with name frameLayoutTop inside of frame of frameLayoutTop should be CardView with name TransferFundsOtherAccountCardView
  9. Inside of a CardView TransferFundsOtherAccountCardView add TextView with name transferFundsCardNameOtherAccountTextView . This TextView will contain hint "Card account name" . Below TextView and also inside of a CardView TransferFundsOtherAccountCardView put TextView with name transferFundsCardAccountNumberOtherAccountTextView with hint SI65463646432.
  10. Next component is of type SwitchCompat, with name transferFundsSwitch. You can add it below the FrameLayout frameLayoutTop in the fragmentTransferFunds. This switch we will later use for switching beetwen transfer accounts.
  11. Next add at the left middle side of the fragmentTransferFunds below textview Transfer from, TextView with name transferFundsToAccountTextView with text Transfer to
  12. On the right side of this Transfer to textview and bellow transferFundsSwitch add second FrameLayout named frameLayoutBottom.
  13. Then create second CardView with name TransferFundsUserAccountCardView and inside it add TextView with name transferFundsCardNameTextView including hint:Card account name user. Below it and also inside of this cardView add TextView named transferFundsCardAccountNumberTextView with hint SI45757699497
  14. Next add to fragmentTransferFunds below second TransferFundsUserAccountCardView, EditText with name transferFundsEnterAmountEditText and hint Enter amount.
  15. And last component to add to fragmentTransferFunds is of type Button with name transferFundsButton. Add it below transferFundsEnterAmountEditText.
  16. After successful login, user with a click on a button named View Balance will navigate user to View balance screen. To make the navigation you can use findNavController().navigate(R.id.action_userMenuFragment_to_viewBalanceFragment). It is also possible to pass a Bundle as second argument of navigate method to pass arguments to viewBalanceFragment, which you can later retrieve with arguments?.getString(...)
    17. After successful login, user with a click on a button Transfer funds will navigate user to Transfer funds screen. It is also possible to pass a Bundle as second argument of navigate method to pass arguments to transferFundsFragment, which you can later retrieve with arguments?.getString(...)
  17. All requirements from stage1 and stage2 are still valid, including toast messages
