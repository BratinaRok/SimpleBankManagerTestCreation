Description

At last stage we will implement a feature to pay bills with our bank manager application. After user logs in and goes to pay bills screen, there user should input bill code and with correct code, information about bill will be shown in dialog. User should have option to choose if confirms bill payment and if has enough balance in account, balance should be subtracted from current balance and bill should be successfully paid.


Objectives

  1. Add a Fragment named PayBillsFragment to the navigation graph;
    • Add an EditText to PayBillsFragment with id PayBillsCodeInputEditText, the hint should be Enter code;
    • Add an Button to PayBillsFragment with id PayBillsShowBillInfoButton and with a text Show bill info;
  2. On the design tab of nav_graph make a connection from UserMenuFragment to PayBillsFragment. The code generated after making the connection should contain an action with id action_userMenuFragment_to_payBillsFragment and destination payBillsFragment;
  3. After click on userMenuPayBillsButton button, display PayBillsFragment. There user should input bill code and click on PayBillsShowBillInfoButton, dialog should appear. If user enters correct bill code dialog of type AlertDialog with title Bill info,and message(each bill info in new line) : Name:(bill name) BillCode:(bill code) Amount:(bill amount) and after info two buttons with text Confirm and Cancel. if user click on Confirm button and there is enough balance in account, bill amount should be subtracted from this account balance and Toast message appears with text "Payment for bill "bill name", was successful" and then dialog should close. If user clicks on Confirm button and there is not enough balance in account, then dialog will appear of type AlertDialog with title Error , message Not enough funds and button with text Ok. If wrong code was entered than dialog of type AlertDialog with title Error, message Wrong code and button text with Ok should appear.
  4. Test may pass the defaultBillInfoMap as a Serializable to MainActivity through intent.extras. Retrieve that defaultBillInfoMap, using the key "billInfo". You will want to cast it back to a map to be able to use it as a map. The type signature of defaultBillInfoMap is Map<String, Triple<String, String, Double>>. In case no defaultBillInfoMap is passed use 

    
             val defaultBillInfoMap =
            mapOf(
                "ELEC" to Triple("Electricity", "ELEC", 45.0),
                "GAS" to Triple("Gas", "GAS", 20.0),
                "WTR" to Triple("Water", "WTR", 25.0)
            )
    as default value for defaultBillInfoMap. The keys are "ELEC", "GAS", and "WTR", and the values are instances of the class Triple. The Triple class is a simple data class that has three properties: a string for the billName, a string for the billCode, and a double for the billAmount.
  5. While at PayBillsFragment clicking on back button should redirect users back to UserMenuFragment;
