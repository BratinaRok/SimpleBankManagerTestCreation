Introduction and basic setup

A bank app is a really common kind of app that most of us use or have used to manage our bank account and pay bills. On this project we will make a simplified version of that kind of application.

This project will have multiple views and we are going to use a single activity pattern, in which the activity will act essentially as a fragments holder and fragments will be responsible for creating screens for users to interact with.

To help us with that we will use androidx.navigation libraries, that will allow us to manage views through a navigation graph.

To use androidx.navigation libraries you have to include them on the build.gradle using: 


    def navigationVersion = '2.4.2'  // or another version number
    implementation "androidx.navigation:navigation-fragment-ktx:$navigationVersion"
    implementation "androidx.navigation:navigation-ui-ktx:$navigationVersion"

But we have already included them, so you should not need to change build.gradle

Setting up a navigation graph is pretty simple using Android Studio. Right click over the res folder and choose New -> Android Resource File. Then name the file, the conventional name is nav_graph, and set Resource Type to Navigation

Adding fragments is also very simple using a navigation graph. Go to the design tab on the nav_graph.xml file, click on New Destination button and then click on Create New Destination. A wizard for creating a new fragment should appear and work as usual.

Objectives

  1. Setup activity_main.xml to hold a NavHostFragment. Use can something like this: 
    <androidx.fragment.app.FragmentContainerView
    android:id="@+id/nav_host_fragment"
    android:name="androidx.navigation.fragment.NavHostFragment"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    app:defaultNavHost="true"
    app:navGraph="@navigation/nav_graph" />
  2. Add a Fragment named LoginFragment to the navigation graph and set it as home
  3. Add an EditText to LoginFragment with id loginUsername, inputType textPersonName, and hint username
  4. Add an EditText to LoginFragment with id loginPassword, inputType numberPassword, and hint password
  5. Add a Button to LoginFragment with id loginButton and text log in
  6. Test may pass a username to MainActivity through the savedInstanceState Bundle argument in OnCreate method. Retrieve that username and use it to validate login. In case no username is passed use "Lara" as default value for username.
  7. Test may pass a password to MainActivity through the savedInstanceState Bundle argument in OnCreate method. Retrieve that password and use it to validate login. In case no password is passed use "1234" as default value for password.
  8. Add a listener to loginButton, when clicked it should do the validation of credentials (username, password) using the values required above
  9. If the validation of credential is successful show a toast with message logged in
  10. If the validation of credential is not successful show a toast with message invalid credentials
  11. Don't use "static" variables to make communication between MainActivity and LoginFragment. Consider "static" variables to be any variable that is either on global scope or inside any singleton object (including companion objects). Besides being a bad pattern tests will also not guarantee to work properly since it uses Junit and Junit does not re-instantiate static variables between test cases as tests expects. You can refer to Communicating to activity item on 'Fragment communication and lifecycle' topic as a possible way to handle this communication
